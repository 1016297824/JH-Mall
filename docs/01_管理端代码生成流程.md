## 一、生成 DDL 脚本

来源：`docs/design/03_系统详细设计.md` 第一章（1.1~1.6 节，共 26 张表）。

- 输出到 `db/mall-sql/` 目录
- 命名：`V1.0.0__create_mall_{service}_tables.sql`
- 按模块先后：mall-user → mall-product → mall-order → mall-payment → mall-marketing

## 二、导入 MySQL

```sql
-- 创建数据库
CREATE DATABASE IF NOT EXISTS mall DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci;

-- 按顺序执行 DDL 脚本（带 --default-character-set=utf8mb4 避免中文乱码）
mysql --default-character-set=utf8mb4 mall < V1.0.0__create_mall_user_tables.sql
mysql --default-character-set=utf8mb4 mall < V1.0.0__create_mall_product_tables.sql
mysql --default-character-set=utf8mb4 mall < V1.0.0__create_mall_order_tables.sql
mysql --default-character-set=utf8mb4 mall < V1.0.0__create_mall_payment_tables.sql
mysql --default-character-set=utf8mb4 mall < V1.0.0__create_mall_marketing_tables.sql
```

代码生成时表导入 `ry-cloud` 库（避免与运行期库混用），DDL 重复执行一次：

```sql
mysql --default-character-set=utf8mb4 ry-cloud < V1.0.0__create_mall_user_tables.sql
mysql --default-character-set=utf8mb4 ry-cloud < V1.0.0__create_mall_product_tables.sql
...
```

## 三、建菜单目录

代码生成前，先在若依管理端构建菜单树。生成的 Menu SQL 会依赖这些目录的 menu_id。

路径：**系统管理 → 菜单管理 → 新增**

### 3.1 顶级目录（已有则跳过）

| 菜单名称 | 路由地址 |   类型    | 排序 |
| -------- | -------- | :-------: | :--: |
| 商城管理 | `mall`   | 目录（M） |  2   |

### 3.2 二级目录

父级菜单 = 商城管理，类型 = 目录：

| 排序 | 菜单名称 | 路由地址         |    图标     |
| :--: | -------- | ---------------- | :---------: |
|  1   | 用户管理 | `mall-user`      |   `user`    |
|  2   | 商品管理 | `mall-product`   |   `goods`   |
|  3   | 订单管理 | `mall-order`     |   `order`   |
|  4   | 支付管理 | `mall-payment`   |   `money`   |
|  5   | 营销管理 | `mall-marketing` | `promotion` |

建完后记下每个目录的 menu_id，后面配置代码生成时作为"上级菜单"使用。`mall-order`、`mall-payment`、`mall-marketing` 三个目录也可等后续生成对应模块时再建。

## 四、若依代码生成

### 4.1 导入前配置

打开若依 Nacos 配置 `ruoyi-gen-dev.yml`，确认数据源指向代码生成库：

```yaml
# ruoyi-gen-dev.yml —— 仅代码生成器使用
# 所有 DDL 表已导入 ry-cloud，生成器从这读表结构
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/ry-cloud?useUnicode=true&characterEncoding=utf8mb4

# 运行期各服务通过 Nacos 各自 DataId（mall-xxx-dev.yml）
# 连接 mall 库，与 ry-cloud 无关
```

### 4.2 导入表

1. 浏览器登录 RuoYi 管理端 → 系统工具 → 代码生成
2. 点击 **导入**，从 `ry-cloud` 库勾选需要生成的 `mall_*` 表
3. 导入后编辑每张表的生成配置：

| 配置项   | 值                 | 说明                                   |
| -------- | ------------------ | -------------------------------------- |
| 表前缀   | `mall_`            | 剥离类名中的 Mall 前缀                 |
| 包名     | `com.mall.${模块}` | 如 `com.mall.user`、`com.mall.product` |
| 模块名   | `${服务名}`        | 如 `mall-user`、`mall-product`         |
| 业务名   | 自动生成，手工校验 | 见 4.2.1 对照表                        |
| 功能名   | 中文描述           | 与表注释一致                           |
| 上级菜单 | 选对应二级目录     | 见 4.2.1 对照表                        |

> **注意**：若两张表的 `business_name` 相同（如 `growth_log` 和 `points_log` 都变成 `log`），生成 ZIP 时文件名会冲突报 `duplicate entry`。导入后在 `gen_table` 表中手工 UPDATE 不同的值，或每张表单独生成。

每导入一张表，在代码生成编辑页同步设好参数后点"保存"。

**操作步骤**（按模块先后，一次生成一整个模块）：

1. 只导入**主表**（如 `mall_product_spu`），子表（`mall_product_sku`）不导入
2. 在生成配置中设"生成模板"为 `sub`，填"关联子表名"和"子表关联外键名"
3. 保存后点"生成代码"，ZIP 中同时包含主表和子表的 Java/Vue 代码
4. 按 4.3 节放置生成的文件

### 4.3 生成代码

点击 **生成代码** 下载 ZIP，解压后按以下结构放置：

```
ZIP 根目录/
├── main/
│   └── java/com/mall/${模块}/          → server/mall/mall-${模块}/src/main/java/com/mall/${模块}/
│       ├── domain/                     → entity（需补 @TableName 注解）
│       ├── mapper/                     → Mapper 接口
│       ├── service/                    → Service + Impl
│       └── controller/                 → Controller
├── resources/
│   └── mapper/${模块}/                 → server/mall/mall-${模块}/src/main/resources/mapper/${模块}/
├── vue/api/${模块}/*.ts                 → client/ruoyi-ui/src/api/${模块}/
├── vue/types/api/${模块}/              → client/ruoyi-ui/src/types/api/${模块}/
├── vue/views/${模块}/                   → client/ruoyi-ui/src/views/${模块}/
└── *Menu.sql                           → 在 ry-cloud 库执行，生成管理端菜单
```

### 4.3.1 合并类型导出入口

ZIP 中 `vue/types/api/` 目录附带一个 `index-bak.ts`，包含当前模块的类型导出声明，
需手动追加到 `client/ruoyi-ui/src/types/api/index.ts`：

1. 打开 `index-bak.ts`，将其中 `export * from "./mall-{模块}/..."` 行复制到
   `index.ts` 末尾
2. **检查同名冲突**：若关联的若依原生模块已有同名导出（如 `system/user` 和
   `mall-user/user` 都导出 `UserQueryParams`），需将冲突行的 `export *` 改
   为显式导出非冲突类型：
   ```typescript
   export type { MallUser } from "./mall-user/user";
   // 其余无冲突的保留 export *
   export * from "./mall-user/address";
   ```
3. 务必在同一 commit 中将 `api/mall-user/user.ts` 等引用此类型文件的导入从
   `@/types`（barrel）改为直接路径：
   ```typescript
   // 改前（走 barrel，同名冲突时拿不到）
   import type { UserQueryParams, MallUser } from "@/types";
   // 改后（走直接路径，唯一确定）
   import type { MallUser } from "@/types";
   import type { UserQueryParams, MallUser } from "@/types/api/mall-user/user";
   ```
4. 追加完成后删除 `index-bak.ts`

### 4.4 放置后处理

1. **DO 类重命名**：若代码生成器带了 `Mall` 前缀（如 `MallUser`），按设计规范建议保留（模块包内不重复包名）；若想去掉，改 `gen_table.class_name` 后重新生成
2. **执行 Menu SQL**：ZIP 中的 `*Menu.sql` 已自带正确的 `parent_id`，在 `ry-cloud` 库执行即可自动挂到对应目录下
3. **清理旧测试**：生成的 `test/` 目录若无用可删除
4. **重启动态**：`ruoyi-system` 服务无需重启，菜单 SQL 立即生效；`mall-*` 服务需重启加载新代码

> **注意**：搜索模块 `mall-search` 不从数据库生成（无 MySQL 表）。

## 五、手写业务逻辑

代码生成器覆盖标准 CRUD，以下需手写：

| 模块           | 手写内容                                                                   | 参考设计文档          |
| -------------- | -------------------------------------------------------------------------- | --------------------- |
| mall-auth      | TokenService、SmsService、WechatAdapter、BCrypt 校验、RemoteUserAdapter    | 06 mall-auth 3.1~3.4  |
| mall-order     | OrderStateMachine、下单编排（幂等/锁库存/锁券/补偿）、Outbox 投递、MQ 消费 | 04 mall-order 5~7     |
| mall-payment   | PaymentStateMachine、渠道适配器（Wechat/Alipay）、回调验签+防重放、Outbox  | 05 mall-payment 3~8   |
| mall-marketing | CouponStateMachine、优惠试算引擎（两阶段过滤+组合择优）、规则匹配          | 07 mall-marketing 3~5 |
| mall-product   | StockServiceImpl（乐观锁库存操作）、SkuCacheService、搜索同步双通道        | 08 mall-product 3~5   |
| mall-search    | ES 索引管理、全文搜索、聚合、降级兜底                                      | 09 mall-search 3~5    |
| mall-user      | 积分操作（乐观锁）、成长值计算、手机号加密查询（SHA256 哈希）              | 10 mall-user 3~5      |
| mall-api       | Feign 接口 + 共享 DTO + 枚举                                               | 11 mall-api 3~7       |
