#### 8.2.2 各模块业务配置（简写）

每个 C 端模块的 `mall-xxx-dev.yml` 均包含 Sentinel 控制台连接与规则持久化配置：

```yaml
spring:
  cloud:
    sentinel:
      eager: true
      transport:
        dashboard: 127.0.0.1:8718
      datasource:
        ds1:
          nacos:
            server-addr: 127.0.0.1:8848
            dataId: sentinel-mall-{module}
            groupId: DEFAULT_GROUP
            data-type: json
            rule-type: flow
```

`{module}` 替换为 `auth` / `user` / `product` / `order` / `payment` / `marketing` / `search`。对应的 Nacos 配置项初始值为 `[]`。

| 配置项 | DataId | 说明 |
| --- | --- | --- |
| `mall.auth.*` | `mall-auth-dev.yml` | access-token-ttl / refresh-token-ttl / sms / wechat 等 |
| `mall.order.*` | `mall-order-dev.yml` | 订单超时、补偿 cron |
| `mall.payment.*` | `mall-payment-dev.yml` | 支付渠道、回调超时 |
| `mall.user.*` | `mall-user-dev.yml` | 用户相关配置 |
| `mall.product.*` | `mall-product-dev.yml` | 商品相关配置 |
| `mall.marketing.*` | `mall-marketing-dev.yml` | 营销相关配置 |
| `mall.search.*` | `mall-search-dev.yml` | 搜索相关配置 |

> `mall.security.jwt-secret` 和 `mall.security.internal-secret` 均在 `application-dev.yml` 中**只配一份**，不再在各模块中重复。

### 8.3 测试环境内存配置

#### 8.3.1 6GB 虚拟机（推荐，全量启动）

| 组件 | 内存 | JVM / 配置 |
| --- | :-----------: | --- |
| MySQL 8.0 | 384MB | `innodb_buffer_pool_size=64M` |
| Redis 7.4 | 128MB | `maxmemory 128mb` |
| RocketMQ NameServer | 128MB | `-Xms128m -Xmx128m` |
| RocketMQ Broker | 384MB | `-Xms256m -Xmx384m` |
| Nacos 3.2 | 256MB | `-Xms256m -Xmx256m`，单机 |
| Elasticsearch 9.x | 512MB | `-Xms512m -Xmx512m`，单节点 |
| Java 服务 × 12 | 1.5GB | 每个 `-Xmx128m` |
| OS + Docker | 500MB | — |
| 余量 | ~2.4GB | |
| **合计** | **6GB** | |

#### 8.3.2 4GB 虚拟机（极限压缩）

| 组件 | 内存 | JVM / 配置 |
| --- | :-------------: | --- |
| MySQL 8.0 | 256MB | `innodb_buffer_pool_size=32M` |
| Redis 7.4 | 128MB | `maxmemory 128mb` |
| RocketMQ NameServer | 128MB | `-Xms128m -Xmx128m` |
| RocketMQ Broker | 256MB | `-Xms128m -Xmx256m` |
| Nacos 3.2 | 256MB | `-Xms256m -Xmx256m`，单机 |
| Elasticsearch 9.x | **384MB** | `-Xms128m -Xmx256m`，`bootstrap.memory_lock: false` |
| Java 服务 × 12 | 1.5GB | 每个 `-Xmx128m` |
| OS + Docker | 500MB | — |
| 余量 | ~600MB | |
| **合计** | **4GB** | |

> **4GB 注意：** ES 极限压缩，仅支持百条级测试数据；出现 OOM 时适当减少索引或调大 `-Xmx`。
