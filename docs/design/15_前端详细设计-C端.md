# JH-Store C 端前端详细设计

> 对应项目 `client/mall-ui/`。管理端 `client/ruoyi-ui/` 基于若依代码生成，不独立设计。

## 一、全局设计

### 1.1 技术栈

| 层       | 选型                                    | 说明                                 |
| -------- | --------------------------------------- | ------------------------------------ |
| 框架     | Vue 3.5 (Composition API)               | 与若依管理端同栈                     |
| 语言     | TypeScript 6.0                          | 全量 TS                              |
| 构建     | Vite 8                                  | 快速冷启动和 HMR                     |
| 路由     | Vue Router 5                            | 导航守卫 + Token 校验                |
| 状态管理 | Pinia 3                                 | 按模块拆分 store                     |
| HTTP     | Axios                                   | 请求拦截注入 Token，响应拦截处理 401 |
| UI 组件  | Element Plus                            | 响应式组件，一套代码覆盖 PC Web + H5 |
| 测试     | Vitest + Playwright                     | 单元测试 + E2E                       |
| 多端     | PC+H5 一套代码，小程序/APP 另建独立项目 | 概要设计 15.3 多端支持原则           |

### 1.6 HTTP 请求层

\`\`\`
client.ts（Axios 实例）：
  请求拦截：
    ① 注入 Authorization: Bearer {accessToken}
    ② 注入 clientRequestNo（下单/支付/领券等写操作，UUID v4）
  响应拦截：
    ① 200：返回 response.data
    ② 401：尝试 refreshToken → 成功替换 accessToken 重试 → 失败跳 /login
    ③ errorCode != 00000：Toast 展示 userTip
\`\`\`

## 二、模块设计

> 以下按后端模块顺序排列，与 \`docs/design/\` 中 08~14 号后端设计文档对齐。

### 2.1 认证模块（auth）—— mall-auth

C 端身份认证中心，负责注册/登录/Token 管理/短信验证码/微信认证/密码管理/账号注销。

#### 页面清单

| 页面     | 路由          | 组件                       |
| -------- | ------------- | -------------------------- |
| 登录     | \`/login\`    | \`LoginPage.vue\`          |
| 注册     | \`/register\` | \`RegisterPage.vue\`       |
| 忘记密码 | \`/forgot\`   | \`ForgotPasswordPage.vue\` |

#### Token 流程

登录成功 → \`store.auth.setToken(accessToken, refreshToken)\` → Axios 请求拦截器注入 \`Authorization\` header → 响应 401 → \`useAuth.refreshToken()\` → 成功替换重试 / 失败跳 \`/login\`

#### API 调用

| 操作       | 后端接口                         |
| ---------- | -------------------------------- |
| 密码登录   | POST `/api/auth/sessions`        |
| 短信登录   | POST `/api/auth/sessions/sms`    |
| 发送验证码 | POST `/api/auth/sms_codes`       |
| 注册       | POST `/api/auth/users`           |
| 刷新 Token | POST `/api/auth/sessions/refresh` |
| 退出登录   | DELETE `/api/auth/sessions/current` |
| 重置密码   | PUT `/api/auth/password/reset`   |
| 修改密码   | PUT `/api/auth/password`         |
| 换绑手机号 | PUT `/api/auth/phone`            |
| 注销账号   | DELETE `/api/auth/account`       |
| 检查会话   | GET `/api/auth/sessions/current` |
| 检查会话   | GET `/api/auth/sessions/current` |

### 2.3 商品模块（product）—— mall-product

C 端商品数据管理，提供类目树/品牌/SPU/SKU 查询、热点商品排行、ES 降级 DB 兜底查询。

#### 页面清单

| 页面     | 路由                | 组件                      |
| -------- | ------------------- | ------------------------- |
| 首页     | \`/\`               | \`HomePage.vue\`          |
| 类目页   | \`/categories\`     | \`CategoryPage.vue\`      |
| 类目商品 | \`/categories/:id\` | \`ProductListPage.vue\`   |
| 商品详情 | \`/products/:id\`   | \`ProductDetailPage.vue\` |

#### SKU 选择器交互

1. 加载 SPU 详情（含 SKU 列表）→ 构建规格矩阵
2. 用户点选规格（颜色/尺寸）→ 匹配 SKU → 显示价格+库存
3. 未选中完整规格 → "加入购物车"按钮灰色
4. 已选中 → 显示实时价格和库存 → 数量选择器
5. 加入购物车 / 立即购买

#### API 调用

| 操作     | 后端接口                          |
| -------- | --------------------------------- |
| 类目树   | GET `/api/product/categories`     |
| 类目详情 | GET `/api/product/categories/{id}` |
| 品牌列表 | GET `/api/product/brands`         |
| SPU 列表 | GET `/api/product/spus`           |
| SPU 详情 | GET `/api/product/spus/{spuId}`   |
| SKU 详情 | GET `/api/product/skus/{skuId}`   |
| 热门商品 | GET `/api/product/spus/hot`       |

### 2.5 订单模块（order）—— mall-order

C 端订单编排核心，管理购物车增删改查、下单流程（幂等/校验/锁库存/锁优惠）、订单状态机（9 状态 12 事件）、售后申请与审核。

#### 页面清单

| 页面     | 路由                       | 组件                    |
| -------- | -------------------------- | ----------------------- |
| 购物车   | \`/cart\`                  | \`CartPage.vue\`        |
| 下单页   | \`/checkout\`              | \`CheckoutPage.vue\`    |
| 订单列表 | \`/orders\`                | \`OrderListPage.vue\`   |
| 订单详情 | \`/orders/:id\`            | \`OrderDetailPage.vue\` |
| 售后申请 | \`/orders/:id/after-sale\` | \`AfterSalePage.vue\`   |

#### 购物车

1. 加载购物车列表（含实时价格+库存状态）
2. 库存不足/已下架商品 → 灰显标注
3. 全选/单选 → 计算合计金额
4. 修改数量 → 乐观更新
5. 删除 → 确认弹窗
6. 结算 → 跳转 \`/checkout\`

#### 下单流程

① 展示选中商品清单 → ② 选择收货地址 → ③ 选择优惠券 → 优惠试算（调用 mall-marketing） → ④ 展示金额明细 → ⑤ 提交订单（POST /api/order/orders，Idempotent-Key） → 成功跳转 \`/payment/:orderNo\`

#### 订单列表

Tab 切换：全部 / 待付款 / 待发货 / 待收货 / 待评价。
按状态展示操作按钮：去支付、取消订单、查看物流、确认收货、申请售后、删除订单。

#### 订单详情

订单号/状态/时间、收货地址、商品列表（图片+名称+规格+单价×数量）、金额明细（总额-优惠-折扣+运费=实付）、物流信息（待发货/待收货可见）。

#### 售后

选择售后类型（仅退款/退货退款）→ 选择商品和数量 → 填写原因 → 提交 → 进度展示（PENDING→APPROVED→REFUNDING→REFUNDED/REJECTED）。

#### API 调用

| 操作       | 后端接口                                  |
| ---------- | ----------------------------------------- |
| 购物车列表 | GET `/api/order/cart/items`               |
| 加入购物车 | POST `/api/order/cart/items`              |
| 修改数量   | PUT `/api/order/cart/items/{id}`          |
| 删除商品   | DELETE `/api/order/cart/items`            |
| 创建订单   | POST `/api/order/orders`                  |
| 订单列表   | GET `/api/order/orders`                   |
| 订单详情   | GET `/api/order/orders/{id}`              |
| 取消订单   | POST `/api/order/orders/{id}/cancellation` |
| 确认收货   | PUT `/api/order/orders/{id}/receipt`      |
| 删除订单   | DELETE `/api/order/orders/{id}`           |
| 售后申请   | POST `/api/order/after_sales`             |
| 售后列表   | GET `/api/order/after_sales`              |

### 2.6 支付模块（payment）—— mall-payment

C 端资金通道服务，负责支付单创建、支付渠道（微信/支付宝）路由与适配、支付回调处理、退款管理。

#### 页面清单

| 页面     | 路由                         | 组件                      |
| -------- | ---------------------------- | ------------------------- |
| 支付页   | \`/payment/:orderNo\`        | \`PaymentPage.vue\`       |
| 支付结果 | \`/payment/result/:orderNo\` | \`PaymentResultPage.vue\` |

#### 支付流程

① 展示订单摘要（金额+倒计时 30min） → ② 选择支付方式（微信/支付宝） → ③ 发起支付获取 payParams → ④ 唤起支付 → ⑤ 轮询支付状态（每 3s） → ⑥ 成功跳转结果页 / 超时显示已关闭

#### API 调用

| 操作       | 后端接口                               |
| ---------- | -------------------------------------- |
| 发起支付   | POST `/api/payment/payments`           |
| 查支付状态 | GET `/api/payment/payments/{paymentId}` |
