# JH-Store C 端前端详细设计

> 对应项目 `client/mall-ui/`。管理端 `client/ruoyi-ui/` 基于若依代码生成，不独立设计。

## 一、全局设计

### 1.1 技术栈

| 层       | 选型                                    | 说明                                 |
| -------- | --------------------------------------- | ------------------------------------ |
| 框架     | Vue 3.5 (Composition API)               | 与若依管理端同栈                     |
| 语言     | TypeScript 6.0                          | 全量 TS                              |
| 构建     | Vite 8                                  | 快速冷启动和 HMR                     |
| 路由     | Vue Router 5                            | 导航守卫 + Token 校验                |
| 状态管理 | Pinia 3                                 | 按模块拆分 store                     |
| HTTP     | Axios                                   | 请求拦截注入 Token，响应拦截处理 401 |
| UI 组件  | Element Plus                            | 响应式组件，一套代码覆盖 PC Web + H5 |
| 测试     | Vitest + Playwright                     | 单元测试 + E2E                       |
| 多端     | PC+H5 一套代码，小程序/APP 另建独立项目 | 概要设计 15.3 多端支持原则           |

### 1.5 状态管理

| Store       | 核心字段                                                       |        持久化        |
| ----------- | -------------------------------------------------------------- | :-------------------: |
| `auth`    | `accessToken`, `refreshToken`, `userId`, `userProfile` |     localStorage     |
| `cart`    | `items[]`, `totalCount`, `selectedItems[]`               | 服务端同步 + 本地缓存 |
| `order`   | `currentOrder`（下单临时数据）                               |          否          |
| `address` | `defaultAddress`, `list[]`                                 |          否          |

### 1.6 HTTP 请求层

```
client.ts（Axios 实例）：
  请求拦截：
    ① 注入 Authorization: Bearer {accessToken}
    ② 注入 clientRequestNo（下单/支付/领券等写操作，UUID v4）
  响应拦截：
    ① 200：返回 response.data
    ② 401：尝试 refreshToken → 成功替换 accessToken 重试 → 失败跳 /login
    ③ errorCode != 00000：Toast 展示 userTip
```

## 二、模块设计

### 2.1 认证模块（auth）

#### 页面清单

| 页面     | 路由          | 组件                                                  |
| -------- | ------------- | ----------------------------------------------------- |
| 登录     | `/login`    | `LoginPage.vue`（Tabs：密码登录/短信登录/微信登录） |
| 注册     | `/register` | `RegisterPage.vue`（手机号+验证码+密码+隐私协议）   |
| 忘记密码 | `/forgot`   | `ForgotPasswordPage.vue`（手机号+验证码+新密码）    |

#### Token 流程

```

登录成功 → store.auth.setToken(accessToken, refreshToken)
  → Axios 请求拦截器自动注入 Authorization header
  → 响应 401 → useAuth.refreshToken()
    ├─ 成功 → 替换 token → 重试原请求
    └─ 失败 → 清除 token → router.push('/login')

```

#### API 调用

| 操作       | API                                                              | 方法        |
| ---------- | ---------------------------------------------------------------- | ----------- |
| 密码登录   | `auth.loginByPassword({ phone, password })`                    | api/auth.ts |
| 短信登录   | `auth.loginBySms({ phone, smsCode })`                          | api/auth.ts |
| 微信登录   | `auth.loginByWechat({ wechatCode, scene })`                    | api/auth.ts |
| 发送验证码 | `auth.sendSmsCode({ phone, scene })`                           | api/auth.ts |
| 注册       | `auth.register({ phone, password, smsCode, isPrivacyAgreed })` | api/auth.ts |
| 重置密码   | `auth.resetPassword({ phone, smsCode, newPassword })`          | api/auth.ts |
| 退出登录   | `auth.logout()`                                                | api/auth.ts |
| 刷新 Token | `auth.refreshToken({ refreshToken })`                          | api/auth.ts |

#### 下单流程

```

CheckoutPage.vue:

```

```

```

#### OrderListPage.vue

```

Tab 切换：全部 / 待付款 / 待发货 / 待收货 / 待评价
列表项：订单号 + 商品缩略图 + 金额 + 状态 + 操作按钮
操作按钮（按状态）：
  WAIT_PAY     → [去支付] [取消订单]
  PAID         → 等待发货（无按钮）
  WAIT_DELIVER → 查看物流
  WAIT_RECEIVE → [确认收货] [查看物流] [申请售后]
  COMPLETED    → [申请售后] [删除订单]
  CANCELLED    → [删除订单]
  CLOSED       → [删除订单]

```

#### OrderDetailPage.vue

```

订单信息：订单号 / 状态 / 时间
收货地址：姓名 + 手机 + 地址
商品列表：图片 + 名称 + 规格 + 单价 × 数量
金额明细：总额 - 优惠 - 会员折扣 + 运费 = 实付
物流信息：快递公司 + 运单号（WAIT_DELIVER/WAIT_RECEIVE 可见）
操作栏：同 OrderList 操作按钮

```

#### 售后

```

AfterSalePage.vue:
  ① 选择售后类型（仅退款/退货退款）
  ② 选择退款商品和数量
  ③ 填写退款原因
  ④ POST /api/order/after_sales → 创建售后单
  ⑤ 售后进度在订单详情页展示（PENDING → APPROVED → REFUNDING → REFUNDED / REJECTED）

```
