# 系统管理功能

系统管理是若依最核心的模块，涵盖 RBAC 权限模型的完整实现，包括用户管理、角色管理、菜单管理、部门管理、岗位管理。

## 1. 用户管理

### 功能作用
管理系统用户，支持增删改查、导入导出、重置密码、分配角色、状态控制。

### 页面路径
`/system/user` — `views/system/user/index.vue`

### 权限标识
| 操作 | 权限标识 |
|------|----------|
| 查询 | `system:user:list` |
| 新增 | `system:user:add` |
| 修改 | `system:user:edit` |
| 删除 | `system:user:remove` |
| 导出 | `system:user:export` |
| 导入 | `system:user:import` |
| 重置密码 | `system:user:resetPwd` |

### 后端 API
| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/user/list` | 分页查询用户列表 |
| GET | `/user/getInfo` | 获取当前登录用户信息 |
| GET | `/user/{userId}` | 获取指定用户详情 |
| POST | `/user` | 新增用户 |
| PUT | `/user` | 修改用户 |
| DELETE | `/user/{userIds}` | 删除用户 |
| PUT | `/user/resetPwd` | 重置密码 |
| PUT | `/user/changeStatus` | 修改状态 |
| PUT | `/user/authRole` | 分配角色 |
| POST | `/user/export` | 导出 Excel |
| POST | `/user/importData` | 导入用户 |
| POST | `/user/importTemplate` | 下载导入模板 |

### 实现流程（新增用户）
1. 前端打开弹窗，调用 `GET /user` 获取岗位/角色选项
2. 填写表单，前端校验用户名唯一、手机号格式、邮箱格式
3. 提交 `POST /user` 到后端
4. 后端 `SysUserController.add()` 校验：
   - 数据权限（部门、角色）
   - 用户名唯一性
   - 手机号 / 邮箱唯一性
5. 密码 BCrypt 加密，写入 `sys_user` 表
6. 写入 `sys_user_post`（用户岗位关联）和 `sys_user_role`（用户角色关联）

### 功能特色
- **部门树侧边栏**：左侧显示部门树，点击过滤用户列表
- **导入导出**：基于 Apache POI 的注解式 Excel
- **密码策略**：支持初始密码修改提醒、密码过期检测
- **数据权限**：根据角色数据权限范围过滤用户

## 2. 角色管理

### 功能作用
管理角色信息，支持角色-菜单/部门权限分配，控制数据权限范围。

### 页面路径
`/system/role` — `views/system/role/index.vue`

### 权限标识
| 操作 | 权限标识 |
|------|----------|
| 查询 | `system:role:list` |
| 新增 | `system:role:add` |
| 修改 | `system:role:edit` |
| 删除 | `system:role:remove` |
| 导出 | `system:role:export` |

### 后端 API
| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/role/list` | 分页查询角色列表 |
| GET | `/role/{roleId}` | 获取角色详情 |
| POST | `/role` | 新增角色 |
| PUT | `/role` | 修改角色 |
| DELETE | `/role/{roleIds}` | 删除角色 |
| PUT | `/role/dataScope` | 修改数据权限 |
| PUT | `/role/changeStatus` | 状态修改 |
| GET | `/role/authUser/allocatedList` | 已分配用户列表 |
| GET | `/role/authUser/unallocatedList` | 未分配用户列表 |
| PUT | `/role/authUser/selectAll` | 批量授权 |

### 数据权限范围
| 范围值 | 说明 |
|--------|------|
| 1 | 仅本人数据 |
| 2 | 本部门数据 |
| 3 | 本部门及以下数据 |
| 4 | 仅本人数据 |
| 5 | 全部数据 |

### 实现流程（角色授权菜单）
1. 编辑角色时加载 `GET /menu/roleMenuTreeselect/{roleId}` 获取全部菜单树 + 已勾选菜单
2. 提交菜单权限，写入 `sys_role_menu` 表（先删后插）
3. 清理该角色所有用户的 Redis 权限缓存（`menu_` + userId）

## 3. 菜单管理

### 功能作用
管理系统菜单和权限标识，构造前端路由树。

### 页面路径
`/system/menu` — `views/system/menu/index.vue`

### 权限标识
| 操作 | 权限标识 |
|------|----------|
| 查询 | `system:menu:list` |
| 新增 | `system:menu:add` |
| 修改 | `system:menu:edit` |
| 删除 | `system:menu:remove` |

### 后端 API
| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/menu/list` | 菜单列表（树形） |
| GET | `/menu/{menuId}` | 菜单详情 |
| POST | `/menu` | 新增菜单 |
| PUT | `/menu` | 修改菜单 |
| DELETE | `/menu/{menuId}` | 删除菜单 |
| GET | `/menu/getRouters` | 获取当前用户路由 |

### 菜单类型
| 类型 | 说明 |
|------|------|
| M | 目录（含子菜单） |
| C | 菜单（叶子节点，对应页面） |
| F | 按钮（权限标识） |

### 前端动态路由流程
1. 用户登录成功 → 调用 `GET /menu/getRouters`
2. 后端构建路由树（`RouterVo`），含 `path/component/name/meta`
3. 前端 `permission.js` 的 `filterAsyncRouter()` 递归将路由字符串转组件对象
4. `router.addRoutes()` 动态注入路由

## 4. 部门管理

### 功能作用
管理系统组织架构，树形结构展示。

### 页面路径
`/system/dept` — `views/system/dept/index.vue`

### 后端 API
| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/dept/list` | 部门列表（树形） |
| GET | `/dept/{deptId}` | 部门详情 |
| POST | `/dept` | 新增部门 |
| PUT | `/dept` | 修改部门 |
| DELETE | `/dept/{deptId}` | 删除部门 |

### 实现要点
- 祖先关系通过 `ancestors` 字段维护
- 删除时校验是否存在子部门或关联用户
- 数据权限过滤部门列表

## 5. 岗位管理

### 功能作用
管理用户岗位信息，一个用户可拥有多个岗位。

### 页面路径
`/system/post` — `views/system/post/index.vue`

### 后端 API
| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/post/list` | 岗位列表 |
| GET | `/post/{postId}` | 岗位详情 |
| POST | `/post` | 新增岗位 |
| PUT | `/post` | 修改岗位 |
| DELETE | `/post/{postIds}` | 删除岗位 |
| GET | `/post/optionselect` | 岗位选择框列表 |

### 注意事项
- 删除岗位前需校验是否有用户关联该岗位（`sys_user_post` 表）
- 岗位本身不参与权限控制，主要作为用户属性
