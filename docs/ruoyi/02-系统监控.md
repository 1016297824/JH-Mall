# 系统监控功能

系统监控涵盖在线用户、操作日志、登录日志和定时任务。

## 1. 在线用户

### 功能作用
查看当前系统中登录的用户，支持强制下线。

### 页面路径
`/monitor/online` — `views/monitor/online/index.vue`

### 后端 API
| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/online/list` | 在线用户列表 |
| DELETE | `/online/{tokenId}` | 强退用户 |

### 实现流程
- 在线用户数据来源于 Redis 中的 `login_tokens:` 前缀缓存
- 用户登录时，`TokenService` 将用户信息缓存到 Redis（含用户信息、角色、权限）
- 强退操作删除 Redis 缓存即可终止登录态

## 2. 操作日志

### 功能作用
记录用户的操作行为，支持查询和导出。

### 页面路径
`/monitor/operlog` — `views/monitor/operlog/index.vue`

### 后端 API
| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/operlog/list` | 操作日志列表 |
| POST | `/operlog/export` | 导出操作日志 |
| DELETE | `/operlog/{operIds}` | 删除日志 |
| DELETE | `/operlog/clean` | 清空日志 |

### 注解式日志记录
使用 `@Log` 注解自动记录操作日志：

```java
@Log(title = "用户管理", businessType = BusinessType.INSERT)
public AjaxResult add(@Validated @RequestBody SysUser user) { ... }
```

### 实现流程
1. `@Log` 注解标记在 Controller 方法上
2. `LogAspect` 切面拦截，构造 `SysOperLog` 对象
3. 异步调用 `AsyncLogService.recordOperLog()` 保存
4. 日志内容包含：请求 URL、方法、参数、耗时、操作人、IP、是否成功

### BusinessType 枚举
| 类型 | 说明 |
|------|------|
| INSERT | 新增 |
| UPDATE | 修改 |
| DELETE | 删除 |
| GRANT | 授权 |
| EXPORT | 导出 |
| IMPORT | 导入 |
| FORCE | 强退 |
| GENCODE | 生成代码 |
| CLEAN | 清空 |

## 3. 登录日志

### 功能作用
记录用户登录/退出历史，支持账户解锁。

### 页面路径
`/monitor/logininfor` — `views/system/logininfor/index.vue`

### 后端 API
| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/logininfor/list` | 登录日志列表 |
| POST | `/logininfor/export` | 导出登录日志 |
| DELETE | `/logininfor/{infoIds}` | 删除日志 |
| DELETE | `/logininfor/clean` | 清空日志 |
| GET | `/logininfor/unlock/{userName}` | 账户解锁 |

### 实现流程
- 登录成功/失败均通过 `SysRecordLogService` 异步记录到 `sys_logininfor` 表
- 密码错误超过 5 次锁定账户 10 分钟（Redis 中 `pwd_err_cnt:` 缓存）
- 管理员可在页面手动解锁被锁定的账户

## 4. 定时任务

### 功能作用
基于 Quartz 的任务调度管理，支持动态创建、修改、删除、执行任务。

### 页面路径
`/monitor/job` — `views/monitor/job/index.vue`

### 后端 API
| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/job/list` | 任务列表 |
| POST | `/job` | 新增任务 |
| PUT | `/job` | 修改任务 |
| PUT | `/job/changeStatus` | 暂停/恢复 |
| PUT | `/job/run` | 立即执行一次 |
| DELETE | `/job/{jobIds}` | 删除任务 |
| GET | `/job/log/list` | 调度日志列表 |
| DELETE | `/job/log/clean` | 清空调度日志 |

### 实现流程
1. 新增任务时，指定调用目标字符串（如 `ryTask.ryParams('参数')`）
2. 通过反射 + Spring 上下文获取 Bean 并调用方法
3. Quartz 自动触发生成的 Job 类（`AbstractQuartzJob`）
4. 执行完成后记录 `SysJobLog` 调度日志

### 任务调用方式
```
ryTask.ryNoParams()         — 无参方法
ryTask.ryParams('params')   — 含参方法
```

### 内置任务示例
```java
@Component("ryTask")
public class RyTask {
    public void ryNoParams() {
        // 定时执行逻辑
    }
    public void ryParams(String params) {
        // 带参数定时执行逻辑
    }
}
```

### 调度日志
| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/job/log/list` | 调度日志列表 |
| POST | `/job/log/export` | 导出调度日志 |
| DELETE | `/job/log/{jobLogIds}` | 删除日志 |
| DELETE | `/job/log/clean` | 清空日志 |

### Cron 表达式工具
内置 `CronUtils` 工具类，支持：
- Cron 表达式合法性校验
- 获取下次执行时间
- 生成最近 N 次执行时间列表
