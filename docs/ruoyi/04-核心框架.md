# 核心框架功能

核心框架包括网关、认证、公共模块，构成微服务架构的基础设施。

## 1. API 网关（ruoyi-gateway）

### 功能作用
Spring Cloud Gateway 实现，作为系统统一入口，负责请求路由、鉴权、限流、XSS 防护。

### 端口
8080

### 核心组件

#### AuthFilter — 全局鉴权过滤器
```
请求 → 是否白名单？ → 是 → 放行
    → 否 → 获取 Token → 解析 JWT → 校验 Redis 登录态 → 附加用户信息 Header → 放行
```

#### ValidateCodeFilter — 验证码过滤器
- 对登录请求 `/auth/login` 进行验证码校验
- 验证码由 `ValidateCodeHandler` 生成并存储在 Redis 中

#### ValidateCodeHandler — 验证码处理器
- 路由 `/code` 生成图片验证码
- 返回 base64 图片 + uuid，uuid 作为 Redis key

#### CaptchaConfig — 验证码配置
- 使用 Google Kaptcha 生成验证码图片
- 可配置：长度、字体、颜色、干扰线等

#### IgnoreWhiteProperties — 白名单配置
在 Nacos 配置中心配置无需认证的路径：
```yaml
ignore:
  whites:
    - /auth/login
    - /auth/register
    - /code
    - /actuator/**
    - /swagger-ui/**
```

### 网关路由配置
在 Nacos 配置中心中定义各服务的路由规则，将请求 path 映射到对应微服务。

## 2. 认证授权（ruoyi-auth）

### 功能作用
处理登录、注册、退出、令牌刷新、解锁屏幕等认证相关操作。

### 端口
9200

### API 列表
| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/auth/login` | 登录 |
| DELETE | `/auth/logout` | 退出 |
| POST | `/auth/refresh` | 刷新令牌 |
| POST | `/auth/register` | 注册 |
| POST | `/auth/unlockscreen` | 解锁屏幕 |

### 登录完整流程
```
[前端] → 输入账号密码 + 验证码
                              ↓
[网关 ValidateCodeFilter] → 校验验证码
                              ↓
[Auth POST /login] → SysLoginService.login()
  ├─ 参数校验（空值、长度、IP 黑名单）
  ├─ RemoteUserService.getUserInfo()（Feign 调用 system 服务）
  │   └─ 获取用户信息 + 角色集合 + 权限集合
  ├─ 用户状态校验（已删除/已停用）
  ├─ SysPasswordService.validate()
  │   ├─ 密码错误次数校验（Redis 累计）
  │   └─ BCryptPasswordEncoder.matches()
  ├─ 记录登录日志（异步）
  ├─ 更新用户登录 IP/时间
  └─ TokenService.createToken()
      └─ 生成 JWT Token + 用户信息缓存 Redis
```

### TokenService — 令牌管理
```
createToken(LoginUser)
  ├─ 生成随机 UUID（userKey）
  ├─ 构造 JWT（含 userId + userName + userKey）
  ├─ 用户信息 + 角色 + 权限 → Redis 缓存（login_tokens: + userKey）
  └─ 返回 access_token / expires_in

getLoginUser(HttpServletRequest)
  ├─ 从 Header 获取 Token
  ├─ JWT 解析 → userKey
  └─ Redis 获取用户信息 → 刷新有效期

refreshToken(LoginUser)
  └─ 刷新 Redis 中用户信息的过期时间
```

### 密码错误锁定机制
```
SysPasswordService
  ├─ Redis Key: pwd_err_cnt:{username}
  ├─ 错误阈值：5 次（可配置）
  ├─ 锁定时长：10 分钟（可配置）
  └─ 登录成功后清除计数
```

## 3. 公共模块（ruoyi-common）

### ruoyi-common-core

#### 统一响应体
```java
R<T>        — Feign 调用返回（含 code / msg / data）
AjaxResult  — 内部 Controller 返回（含 code / msg / data）
TableDataInfo — 分页返回（含 rows / total）
```

#### 分页工具
- `BaseController.startPage()` — 从请求参数获取 pageNum/pageSize，自动拦截 MyBatis 分页
- `PageDomain` / `TableSupport` — 分页参数封装
- 基于 PageHelper 实现

#### Excel 工具
- `@Excel` 注解标记字段（列名、排序、类型、字典等）
- `ExcelUtil` 实现导入导出，支持字典映射、数据校验

#### 异常体系
| 异常类 | 用途 |
|--------|------|
| ServiceException | 业务异常 |
| DemoModeException | 演示模式限制 |
| CaptchaException | 验证码错误 |
| InnerAuthException | 内部认证失败 |
| FileException | 文件相关异常 |
| NotLoginException | 未登录 |
| NotPermissionException | 无权限 |
| NotRoleException | 无角色 |

#### 常量类
| 类名 | 说明 |
|------|------|
| Constants | 通用常量 |
| HttpStatus | HTTP 状态码 |
| SecurityConstants | 安全相关常量（Header 头、Token 类型等） |
| CacheConstants | 缓存 Key 前缀 |
| UserConstants | 用户相关常量 |
| TokenConstants | Token 常量 |
| ServiceNameConstants | 微服务名称 |
| GenConstants | 代码生成常量 |
| ScheduleConstants | 定时任务常量 |

### ruoyi-common-security

#### 注解鉴权
| 注解 | 作用 |
|------|------|
| `@RequiresLogin` | 需要登录 |
| `@RequiresPermissions("system:user:list")` | 需要指定权限 |
| `@RequiresRoles("admin")` | 需要指定角色 |
| `@InnerAuth` | 仅内部 Feign 调用 |
| `@EnableCustomConfig` | 启用自定义配置 |
| `@EnableRyFeignClients` | 启用 RuoYi Feign 客户端 |

#### Feign 请求头拦截器
`FeignRequestInterceptor` 自动为 Feign 请求添加 Header：
```
Authorization: Bearer {token}
From-Source: inner-feign（标记内部调用）
```

#### 安全工具
```java
SecurityUtils.getLoginUser()    // 获取当前登录用户
SecurityUtils.getUserId()       // 获取用户 ID
SecurityUtils.getUsername()     // 获取用户名
SecurityUtils.getUserKey()      // 获取用户 Key
SecurityUtils.isAdmin()         // 是否为管理员
SecurityUtils.encryptPassword() // BCrypt 加密
SecurityUtils.matchesPassword() // 密码校验
```

### ruoyi-common-log

#### @Log 注解
```java
@Log(title = "用户管理", businessType = BusinessType.INSERT, operatorType = OperatorType.MANAGE, isSaveRequestData = true)
```

- `LogAspect` 切面拦截，构造操作日志对象
- `AsyncLogService` 异步持久化到数据库

### ruoyi-common-datascope

#### @DataScope 注解
- 在 Mapper 方法上标记数据权限
- `DataScopeAspect` 自动拼接 SQL 过滤条件（部门表别名、用户表别名）
- 根据角色数据权限范围动态拼接 `WHERE` 条件

### ruoyi-common-sensitive

#### @Sensitive 注解
- 在实体字段上标记敏感信息类型
- Jackson 序列化时自动脱敏

| 脱敏类型 | 示例 |
|----------|------|
| 手机号 | 138****1234 |
| 邮箱 | z***@example.com |
| 身份证 | 110***********1234 |
| 银行卡 | 6222****1234 |
| 地址 | 北京市*** |

### ruoyi-common-redis

#### RedisService
Redis 操作统一封装：
- 字符串/哈希/列表/集合/有序集合操作
- 缓存过期设置
- 分布式锁（`tryLock` / `unLock`）
- 自增/自减
- FastJson2 序列化

## 4. 系统接口（ruoyi-api）

### 功能作用
定义微服务间的 Feign 远程调用接口、共享实体和降级工厂。

### RemoteUserService — 用户远程调用
```java
@GetMapping("/user/info/{username}")      // 获取用户信息
@PostMapping("/user/register")            // 注册
@PutMapping("/user/recordlogin")          // 记录登录
```

### RemoteLogService — 日志远程调用
```java
@PostMapping("/operlog")                  // 保存操作日志
@PostMapping("/logininfor")               // 保存登录日志
```

### RemoteFileService — 文件远程调用
```java
@PostMapping("/upload")                   // 文件上传
@DeleteMapping("/delete")                 // 文件删除
```

### 降级工厂
每个 Feign 接口对应一个 FallbackFactory，实现熔断降级逻辑。
