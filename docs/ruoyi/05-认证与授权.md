# 认证与授权

## 一、认证流程总览

```
┌──────────┐     ┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│  前端     │     │  网关 Gateway │     │  Auth 认证    │     │  System 服务  │
│ (Login.vue) │ ──→ │ (AuthFilter)  │ ──→ │ (TokenController) │ ──→ │ (UserController) │
└──────────┘     └──────────────┘     └──────────────┘     └──────────────┘
     │                  │                    │                     │
     │ 1. 登录请求       │                    │                     │
     │ POST /auth/login  │                    │                     │
     │ 含 username/      │                    │                     │
     │ password/code/uuid │                    │                     │
     │──────────────────→│                    │                     │
     │                  │ 2. 验证码校验        │                     │
     │                  │ (ValidateCodeFilter)│                     │
     │                  │────────────────────→│                     │
     │                  │ 3. 转发到 Auth      │                     │
     │                  │──────────────────────────────────────────→│
     │                  │                    │ 4. Feign 调用获取用户   │
     │                  │                    │ GET /user/info/{name} │
     │                  │                    │──────────────────→  │
     │                  │                    │                     │
     │                  │                    │ 5. 返回 LoginUser   │
     │                  │                    │←──────────────────  │
     │                  │                    │                     │
     │                  │                    │ 6. 校验密码/状态    │
     │                  │                    │ 7. 生成 Token       │
     │                  │                    │ 8. Redis 缓存登录态  │
     │                  │                    │                     │
     │ 9. 返回 Token    │                    │                     │
     │←─────────────────│                    │                     │
     │                  │                    │                     │
     │ 10. 存储 Token   │                    │                     │
     │ 到 Cookie/Storage│                    │                     │
     │                  │                    │                     │
     │ 11. 获取路由     │                    │                     │
     │ GET /menu/get    │                    │                     │
     │ Routers          │                    │                     │
     │─────────────────────────────────────→│                     │
```

## 二、Token 结构

### JWT 载荷
```json
{
  "userId": 1,
  "userKey": "a1b2c3d4-e5f6-...",
  "userName": "admin"
}
```

### Redis 缓存结构
```
Key:   login_tokens:{userKey}
Value: LoginUser（用户信息 + 角色集合 + 权限集合）
TTL:   30 分钟（每次请求自动续期）
```

## 三、权限校验机制

### 前端权限控制

| 方式 | 实现 | 说明 |
|------|------|------|
| 路由级 | `router.addRoutes()` | 后端返回路由树，前端动态添加 |
| 按钮级 | `v-hasPermi` 指令 | 根据用户权限集合控制显示 |
| 角色级 | `v-hasRole` 指令 | 根据用户角色集合控制显示 |

### 按钮权限指令示例
```html
<el-button v-hasPermi="['system:user:add']">新增</el-button>
<el-button v-hasRole="['admin']">管理员操作</el-button>
```

### 后端权限控制

| 方式 | 注解 | 说明 |
|------|------|------|
| 权限 | `@RequiresPermissions("system:user:list")` | 必须有指定权限 |
| 角色 | `@RequiresRoles("admin")` | 必须有指定角色 |
| 登录 | `@RequiresLogin` | 必须登录 |

### 内部认证
- `@InnerAuth` 注解标记的方法仅允许 Feign 内部调用
- 通过 Header 中的 `From-Source: inner-feign` 区分
- 请求来源不是 `inner-feign` 则返回 401

## 四、数据权限实现

### @DataScope 注解
```java
@DataScope(deptAlias = "d", userAlias = "u")
public List<SysUser> selectUserList(SysUser user) { ... }
```

### 数据权限范围
| 范围 | SQL 拼接 |
|------|----------|
| 仅本人 | `AND u.user_id = {currentUserId}` |
| 本部门 | `AND d.dept_id = {currentDeptId}` |
| 本部门及以下 | `AND (d.dept_id = {currentDeptId} OR d.ancestors LIKE '%{currentDeptId}%')` |
| 全部 | 不拼接 |

## 五、注册功能

### 前端页面
`/register` — `views/register.vue`

### 后端流程
1. 检查系统配置 `sys.account.registerUser` 是否开启注册
2. 校验用户名唯一性
3. 密码 BCrypt 加密
4. 写入用户表（默认状态正常）

## 六、锁屏功能

### 页面
`/lock` — `views/lock.vue`

### 实现
- 前端存储锁定状态到 `lock` 模块（Vuex）
- 解锁时调用 `POST /auth/unlockscreen` 验证密码
- 验证通过后恢复页面
