# 通知公告

## 功能作用
管理系统的通知公告，支持发布、阅读跟踪。

## 页面路径
系统管理 → 通知公告 — `views/system/notice/index.vue`

## 后端 API

### 公告管理
| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/notice/list` | 公告列表 |
| GET | `/notice/{noticeId}` | 公告详情 |
| POST | `/notice` | 新增公告 |
| PUT | `/notice` | 修改公告 |
| DELETE | `/notice/{noticeIds}` | 删除公告 |
| GET | `/notice/listTop` | 首页顶部公告（最多 5 条） |
| POST | `/notice/markRead` | 标记已读 |
| POST | `/notice/markReadAll` | 全部标记已读 |
| GET | `/notice/readUsers/list` | 已读用户列表 |

## 公告类型
| 类型 | 说明 |
|------|------|
| 1 | 通知 |
| 2 | 公告 |

## 实现流程

### 发布公告
1. 管理员新建公告（标题、内容、类型、状态）
2. 状态为"发布"时用户可见
3. 支持 HTML 富文本内容

### 阅读跟踪
1. 用户查看公告详情时自动记录已读
2. `SysNoticeRead` 表记录用户-公告的已读关系
3. 支持按用户查看公告已读状态
4. 首页顶部显示未读公告列表

### 首页展示
- `GET /notice/listTop` 返回最新 5 条未读公告
- 支持点击跳转到公告详情页
- 支持全部标记已读
