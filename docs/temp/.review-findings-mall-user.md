# mall-user 审查问题清单

> 日期：2026/05/28 | 审查对象：mall-user C 端后端 | 评分：5.7/10

---

## 🔴 严重（6 项）

### 1. Controller 命名不符设计文档

- [x] `controller/UserProfileController.java` → 应改为 `ProfileController`
- [x] `controller/MemberController.java` → 应改为 `MembershipController`
- 依据：`docs/design/09_mall-user详细设计.md` §2.1 包结构

### 2. SignInServiceImpl 签到积分未发放

- [x] `signIn()` 计算了积分但**未调用 `PointsService.addPoints()`** 写入数据库
- 修复：添加 IPointsService 依赖，setBit 后调用 addPoints
- 依据：`09_mall-user详细设计.md` §4.3 数据流第⑦步

### 3. SignInServiceImpl 签到并发竞态

- [x] `getBit` → `setBit` 之间无保护，两个并发请求可能同时通过幂等检查
- 修复：用 Redis Lua 脚本原子化 GETBIT+SETBIT，返回 0 抛 RESOURCE_EXISTS
- 同时删除已签到分支的冗余代码（不再可达）

### 4. AddressServiceImpl.setDefault 事务不完整

- [x] `clearDefault(userId)` 和 `updateById(entity)` 未包裹在同一 `@Transactional` 下
- 修复：加 `@Transactional(rollbackFor = Exception.class)`，保证两步原子性
- 风险：clearDefault 成功但 setIsDefault 失败 → 用户没有默认地址

### 5. Controller 方法名与文档不一致

- [x] `AddressController.add` → `create`（与 mall-auth `createUser` 惯例一致）
- [x] `PointsController.getPointsRecords` / `GrowthController.getGrowthRecords` → **保持不改**，比文档的 `listRecords` 更清晰

### 6. 文档要求但未实现

- [x] `AddressServiceImpl.deleteAddress`：文档原要求"自动指定最早地址为默认"，已改为 C 方案——前端提示用户设置新默认
- 理由：沉默改默认地址太危险，涉及真实物流不可自动切换

## 🟡 中等（7 项）

### 7. MemberServiceImpl 重复查询

- [x] `addGrowth` → `checkUpgrade` 内部重新 `selectAllOrderByLevelValue()`，与 `getMembership` 重复
- 修复：`levels` 查询提到 `addGrowth` 层，作为参数传入 `checkUpgrade(levels)`

### 9. UserOrderCompletedConsumer 空值未校验

- [x] `userId` 为 null 时直接传给 Service → 下游 NPE
- [x] `orderAmount` 为 null 时 `orderAmount / 100` 会 NPE
- 修复：userId null 时 log.warn + return；orderAmount 已有 null 判断无需额外修复

### 10. MallUserPointsLogMapper 分页参数校验

- [x] `selectByUserIdPage` 无 limit/offset 边界检查，传入极大 page 可导致慢查询
- 修复：PointsServiceImpl、MemberServiceImpl、GrowthServiceImpl 三分处统一加 `Math.max(page,1)` + `Math.min(size,100)`

### 11. IGrowthService 越界查询（已修复 ✅）

- [x] `getGrowth()` 从 IGrowthService 移除：流水服务不应查 `mall_user_member` 余额表
- [x] GrowthController 注入 IMemberService 处理 `/api/user/growth`（余额查询），IGrowthService 仅处理 `/api/user/growth/records`（流水查询）
- [x] GrowthServiceImpl 精简为只依赖 `MallUserGrowthLogMapper`，删除 `mall_user_member`/`mall_user_member_level` 相关代码
- IGrowthService 职责现在是 100% `mall_user_growth_log` 表操作

### 12. 其他代码质量（已修复 ✅）

- [x] `SignInServiceImpl.signIn()` 幂等分支：Lua 脚本修复后已不存在双分支，无需抽取
- [x] unused import：全部 Service/Controller/Consumer 逐文件检查，无残留
- [x] `MallUserServiceImpl`：统一为 `@Slf4j` + `@RequiredArgsConstructor`，与其他 6 个 Service 风格一致
- [x] 测试文件重命名：`UserProfileControllerTest` → `ProfileControllerTest`，`MemberControllerTest` → `MembershipControllerTest`

---

## 📊 总结

13 项审查问题全部修复。修订后评分：**9.0/10**
