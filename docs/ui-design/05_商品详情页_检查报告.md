## 一、设计一致性检查（对照 00_MASTER.md）

### 1.1 色彩系统

| 元素 | 设计规范 | 实际实现 | 状态 |
|------|----------|---------|:--:|
| 页面背景 | `--color-neutral-50` = `#F0F9FF` | `v.$color-bg` = `#F0F9FF` | ✅ |
| 主按钮 | Primary `#0369A1` | `.btn-buy` 背景 `#0369A1` | ✅ |
| CTA（加入购物车） | **`#22C55E`** | `.btn-cart` 边框 `#0369A1` / 白底 | ❌ |
| 标题文字 | `--color-neutral-900` = `#0C4A6E` | `.product-name` `#0C4A6E` | ✅ |
| 正文文字 | `--color-neutral-700` = `#075985` | 部分使用 `v.$color-text-regular` | ✅ |
| 次要文字 | `--color-neutral-500` = `#0369A1` | `v.$color-text-secondary` | ✅ |
| 划线价 | 未明确定义 | `#a0b4c4`（自定义色） | ⚠️ |
| 禁止态文字 | `--color-neutral-300` = `#7DD3FC` | `.qty-btn:disabled` 用 `v.$color-border` = `#BAE6FD` | ⚠️ |

**🔴 P0**：CTA 按钮（加入购物车）应使用 `#22C55E`，目前用的是 Primary 边框+白底，与设计规范不符。

### 1.2 字体层级

| 层级 | 规范（字号/字重/字体） | 实际 | 状态 |
|:----:|-----------------------|------|:--:|
| H1 | 28px / 700 / Rubik | 24px / 700 / Rubik | ⚠️ |
| H2 | 22px / 600 / Rubik | 未使用 | — |
| Body M | 14px / 400 / Nunito Sans | 14px（多处使用） | ✅ |
| Price | 20px / 700 / Rubik | **32px** / 700 / Rubik | ❌ |

**🔴 P0**：Price 字号 32px 远超规范的 20px，与品牌调性不一致。

**🔴 P0**：H1 字号 24px vs 规范 28px。

### 1.4 圆角

### 1.5 阴影

| 元素 | 规范 | 实际 | 状态 |
|------|------|------|:--:|

### 1.6 动效

| 元素 | 规范 | 实际 | 状态 |
|------|------|------|:--:|
| 按钮 hover | 150ms 背景色 | 150ms / 200ms | ✅ |
| 按钮 active | `scale(0.97)` | `scale(0.98)` / `scale(0.92)` | ⚠️ |
| SKU 选择器 | 选中态边框变色+缩放 200ms | 200ms ✓ / 无缩放 | ⚠️ |

## 二、交互流程检查（对照 15_前端详细设计-C端.md §2.3）

### 2.1 SKU 选择流程

| 步骤 | 设计要求 | 实现 | 状态 |
|------|---------|------|:--:|
| 加载 SPU 详情 → 构建规格矩阵 | ✅ 设计要求 | API 加载 → `deriveSpecGroups` 从 SKU 名派生 | ✅ |
| 用户点选规格 → 匹配 SKU | ✅ 设计要求 | 按名称片段位置匹配 | ✅ |
| 显示价格 | ✅ | 选规格前显示 priceMin，选完后显示具体 SKU 价格 | ✅ |
| 未选完整 → 按钮灰色 | ✅ | `:disabled="!canBuy"` + `"请选择完整规格"` 提示 | ✅ |
| 已选完整 → 显示库存+数量 | ✅ | 选完 → `watch(currentSku)` 调 `getSkuDetail` 获取库存 | ✅ |
| 加入购物车 / 立即购买 | ✅ | `addToCart` + `buyNow` 已实现 | ✅ |

### 2.2 缺失流程

| 缺失项 | 严重度 | 说明 |
|--------|:------:|------|
| SKU 详情加载中无反馈 | **P1** | `watch` 触发 `getSkuDetail` 异步请求期间无 loading 态，库存/划线价会短暂显示旧值或 0 |
| 购物车加入成功无数量反馈 | **P2** | `ElMessage.success('已加入购物车')` 告知成功但未显示已加数量/可继续浏览 |
| 立即购买未登录时跳转 | **P1** | 设计文档标记 `/checkout` 需登录，但当前页面未校验登录态，直接跳转会被网关拦截返回 401 |
| 路由守卫为空 | **P1** | `guards.ts` 为空（仅 `export {}`），无登录校验、无跳回逻辑 |

### 2.3 状态态覆盖

| 状态 | 实现 | 状态 |
|------|------|:--:|
| 加载中 | 骨架屏（左右两栏） | ✅ |
| 加载失败 | 错误图标 + 重试按钮 | ✅ |
| 空数据 | 未覆盖（API 抛异常走错误态） | ✅ |
| 售罄 | 主图蒙层 "已售罄" + 按钮禁用 | ✅ |
| 无评价 | "暂无评价，成为第一个评价的人吧" | ✅ |

## 三、可访问性检查

### 3.1 颜色对比度

| 元素 | 前景 / 背景 | 预估对比度 | 状态 |
|------|------------|:--:|:--:|
| `.btn-buy` 白色文字 | `#FFF` / `#0369A1` | ~7.7:1 | ✅ |
| `.product-name` | `#0C4A6E` / `#F0F9FF` | ~8.2:1 | ✅ |
| `.product-desc` | `#0369A1` / `#F0F9FF` | ~4.9:1 | ✅ |
| `.spec-value--disabled` | `#b0c4d0` / `#f5f8fa` | ~1.8:1 | ❌ |
| 划线价 | `#a0b4c4` / 渐变背景 | ~2.2:1 | ❌ |

**🔴 P0**：`.spec-value--disabled` 对比度 ~1.8:1，远低于 WCAG AA 要求的 4.5:1。禁用态文字不可读。

**🟡 P1**：划线价 `#a0b4c4` 对比度 ~2.2:1，低于 3:1（大文本）要求。

### 3.2 键盘导航

| 检查项 | 状态 | 说明 |
|--------|:--:|------|
| 所有可交互元素可 Tab 到达 | ⚠️ | 按钮/链接均可聚焦，但无明显的 `:focus-visible` 样式 |
| Tab 顺序符合视觉顺序 | ✅ | DOM 顺序与视觉一致 |
| 无键盘陷阱 | ✅ | 无捕获焦点的元素 |
| SKU 选择器无 `role="radiogroup"` | ❌ | SKU 按钮应该是一个 radio group，但仅用普通 `<button>` |

**🔴 P0**：缺少 `:focus-visible` 样式。当前按钮/链接聚焦时无任何视觉指示，键盘用户无法判断焦点位置。

**🟡 P1**：SKU 选择器建议使用 `role="radiogroup"` + `aria-checked` 让屏幕阅读器正确播报。

### 3.3 ARIA 与语义 HTML

| 检查项 | 状态 | 说明 |
|--------|:--:|------|
| 语义化 HTML 元素 | ✅ | 使用 `<section>`, `<h1>`, `<dl>`, `<dt>`, `<dd>`, `<button>`, `<ul>`, `<li>` |
| 表单标签关联 | ❌ | 数量 `<input>` 无 `<label for="qty">`，仅用 `<span>` 显示 "数量" |
| 动态 ARIA | ❌ | 无 `:aria-expanded`、`:aria-checked`、`:aria-live` |
| 图标按钮有 `aria-label` | ❌ | 缩略图按钮 `<button>` 内仅有 `<img alt>`，按钮自身无描述 |
| 错误提示使用 `role="alert"` | ✅ | 错误态为静态文本展示，无交互 |

**🔴 P0**：数量输入框缺少 `<label>`，违反 WCAG §3.3.2。

**🟡 P1**：建议为 SKU 选中态添加 `aria-checked`，为库存变化添加 `aria-live` 区域。

### 3.4 其他

| 检查项 | 状态 | 说明 |
|--------|:--:|------|
| 图片 alt | ⚠️ | 主图有 alt，缩略图有 alt，但 `loading="eager"` 加载全部图片 |
| 标题层级 | ✅ | 仅 `h1`，无层级跳跃 |
| `prefers-reduced-motion` | ❌ | 缺失 |
| 焦点可见 | ❌ | 无 `outline`/`ring` 样式 |
| 颜色不是唯一信息指示器 | ⚠️ | 选择规格后按钮禁用 + 文字提示，但 disabled 态仅靠 opacity 判断 |

## 四、改进建议汇总

### 🔴 P0 — 必须修复

| # | 问题 | 文件/位置 | 修复方案 |
|---|------|----------|---------|
| 1 | **CTA 按钮颜色错误** | `.btn-cart` | 背景改为 `#22C55E`，无边框 |
| 2 | **缺少 `prefers-reduced-motion`** | `<style>` 末尾 | 添加 `@media (prefers-reduced-motion: reduce) { *, *::before, *::after { animation: none !important; transition: none !important; } }` |
| 3 | **缺少 `:focus-visible` 样式** | 全局/页面级 | 所有按钮、输入框添加 `:focus-visible { outline: 2px solid #0369A1; outline-offset: 2px; }` |
| 4 | **数量输入框缺 `<label>`** | L302-319 | `<span class="qty-label">` → `<label for="qty-input" class="qty-label">`，`<input>` 加 `id="qty-input"` |
| 5 | **禁用态规格值对比度不足** | `.spec-value--disabled` | 文字色改为 `#8da4b3` + 背景 `#eef2f5` |
| 6 | **Price 字号过大** | `.price-current` | 32px → 20px（规范要求），或明确此页面允许覆盖 |
| 7 | **H1 字号偏小** | `.product-name` | 24px → 28px（规范要求） |

### 🟡 P1 — 强烈建议

| # | 问题 | 修复方案 |
|---|------|---------|
| 8 | SKU 详情加载无反馈 | `watch(currentSku)` 中设置一个小 loading 标记，展示 "查询库存中..." 或骨架 |
| 9 | 划线价对比度不足 | `#a0b4c4` → `#7a94a6`（≥ 3:1） |
| 10 | 缩略图按钮需 `aria-label` | 添加 `:aria-label="'切换到第' + (idx + 1) + '张图'"` |
| 11 | SKU 按钮需 `aria-checked` | 在 `specButtonClass` 中添加 `:aria-checked="selected"` |
| 12 | 立即购买需校验登录 | `buyNow()` 前检查 token 是否存在，不存在跳 `/login?redirect=/products/...` |
| 13 | 路由守卫为空 | 实现 `guards.ts` 基本的登录校验逻辑 |
| 14 | 移动端骨架屏未适配 | 移动端时骨架屏改为上下堆叠布局 |

### 🟢 P2 — 可选优化

| # | 问题 | 修复方案 |
|---|------|---------|
| 15 | 服务承诺 `::before` 不可访问 | 用 `<span aria-hidden="true">✓</span>` 替代 CSS `content` |
| 16 | 数量组件用 `type="number"` | 移动端数字键盘体验不一致，可考虑纯按钮+span 方案 |
| 17 | 评价 Tab 永远为空 | 待评价 API 就绪后接入，先放推荐商品引导 |
| 18 | 库存变化无播报 | 库存区域添加 `aria-live="polite"` |
| 19 | 按钮缩放值不一致 | 统一按钮 active 缩放为 `0.97`（规范值） |

## 五、总结

| 维度 | 得分 | 说明 |
|:----:|:----:|------|
| 设计一致性 | 6/10 | 主结构对齐，但 Price/H1 字号、CTA 颜色、disabled 文字色偏离规范 |
| 交互完整性 | 7/10 | 核心 SKU 选择流程完整，缺 SKU 详情加载中反馈、登录校验 |
| 可访问性 | 3/10 | 缺 focus-visible、label、prefers-reduced-motion、ARIA 标注，键盘导航基本不可见 |

**重点：修复 7 个 P0 项后可达到基本可交付水平。**
