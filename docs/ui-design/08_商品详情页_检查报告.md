| 检查项 | 当前实现 | 结果 |

| SKU 选择器 role | `role="radiogroup"` + `role="radio"` + `:aria-checked` | ✅ |
| 服务承诺装饰符 | `<span aria-hidden="true">✓ </span>` | ✅ |
| inputmode="numeric" | 数量输入框 | ✅ |
| prefers-reduced-motion | **仅在父组件 `ProductDetailPage.vue` 中有** | ⚠️ |

### 3.2 发现项

| # | 严重度 | 问题 | 文件 |
|:--:|:------:|------|------|
| 4 | 🔴 P0 | **`prefers-reduced-motion` 仅父组件生效**，子组件（Gallery/InfoSection/Tabs/SkuSelector）的过渡/动画不受控。子组件的 `scoped` 样式不受父 `scoped` 规则影响 | 全部 4 个子组件 |
| 5 | 🟡 P1 | Tab 按钮缺少 `role="tab"` / `aria-selected` / `role="tablist"` / `aria-controls`，屏幕阅读器无法识别为 Tab 组件 | `ProductTabs.vue` |
| 6 | 🟡 P1 | 切换 Tabs 时内容区无焦点管理，键盘用户 Tab 切换后焦点停留在原按钮，不会自然移到新面板内容 | `ProductTabs.vue` |
| 7 | 🟡 P1 | 商品主图 alt 文本仅为商品名（`alt="华为 Pura 80 Pro"`），未描述图中实际内容（如"商品主图"语义） | `ProductGallery.vue` L4 |
| 8 | 🟢 P2 | 规格选择器按钮无 `tabindex` 管理，键盘导航需逐个遍历所有 disabled 按钮 | `SkuSelector.vue` |

## 4. 优先级汇总

### 🔴 P0（0 项）

上一次报告（#07）P0 已清零，本次发现 1 个新 P0。

| # | 问题 | 建议修复 |
|:--:|------|---------|
| 4 | 子组件缺 `prefers-reduced-motion` | 在 `global.scss` 中添加全局 `@media (prefers-reduced-motion: reduce)` 规则，而非依赖父组件 scoped |

### 🟡 P1（4 项）

| # | 问题 | 建议修复 |
|:--:|------|---------|
| 1 | 骨架屏色值非设计系统 | 改为 `$color-bg-card` (`#E0F2FE`) + `$color-border` (`#BAE6FD`) |
| 5 | Tab 缺 WAI-ARIA | 添加 `role="tablist"` / `role="tab"` / `aria-selected` / `aria-controls` |
| 6 | Tab 切换无焦点管理 | `emit('update:modelValue')` 后用 `nextTick()` focus 面板容器 |
| 7 | 商品主图 alt 不充分 | 改为 `商品名 + ' 主图'` 格式 |

### 🟢 P2（3 项）

| # | 问题 | 建议修复 |
|:--:|------|---------|
| 2 | Mobile 标题 24px vs 28px | ProductInfoSection 响应式中移除 font-size 覆盖 |
| 3 | Tab 切换动画无意义 | 移除 `fadeIn` 动画或用 `opacity` only 替代 |
| 8 | disabled 按钮无 `tabindex="-1"` | 在 SkuSelector 中为 disabled 按钮添加 `:tabindex="-1"` |
