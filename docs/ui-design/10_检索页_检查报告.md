## 2. 交互完整性（对照 `docs/design/15_前端详细设计-C端.md`）

### 2.1 通过项

| 步骤 | 设计要求 | 实现 | 结果 |
|:----:|---------|------|:----:|
| URL 同步 | 从 query 读取 keyword | `route.query.keyword` + `onMounted` | ✅ |
| 全文搜索 | POST `/api/search` | `store.doSearch()` | ✅ |
| 搜索建议 | GET `/api/search/suggest` | `store.fetchSuggestions()` + 200ms 防抖 | ✅ |
| 聚合筛选 | 类目/品牌/价格区间 | `selectedCategoryId` / `selectedBrandId` / `priceMinInput`/`priceMaxInput` | ✅ |
| 排序切换 | 综合/销量/价格升/价格降/新品 | `SORT_OPTIONS` + `el-radio-group` | ✅ |
| 分页加载 | 加载更多 | `store.loadMore()` | ✅ |
| 空结果 | 提示 + 建议 | `el-empty` + "换个关键词试试" | ✅ |
| 加载中 | 骨架屏 | `el-skeleton` × 6 | ✅ |
| 响应式 | PC/Tablet/Mobile | 筛选侧栏 → 抽屉 | ✅ |

### 2.2 发现项

| # | 严重度 | 问题 | 位置 |
|:--:|:------:|------|:----:|
| 7 | 🔴 P0 | **无错误状态处理**：ES 不可用/网络超时时无 Toast 提示，用户看到永久骨架屏或空结果 | `SearchPage.vue` script |
| 8 | 🔴 P0 | **搜索高亮未渲染**：后端返回 `spuNameHighlight` 带 `<em>` 标签，前端直接渲染 `item.spuName`，高亮信息丢失 | L381 |
| 9 | 🟡 P1 | **排序后无滚动锚点**：切换排序 → 重新查询 → 页面停留在原位置，用户应该被带回结果顶部 | L38-41 |
| 10 | 🟡 P1 | **顶部导航栏搜索框与检索页搜索框重复**：之前已修（`BaseNavBar` 加了 `v-if="!isSearchPage"`），需确认确认生效 | `BaseNavBar.vue` |
| 11 | 🟢 P2 | **移动端筛选按钮文案**：`el-button` 写死"筛选"，应包含当前筛选数量（如"筛选(3)"） | L223 |

| 17 | 🟡 P1 | **加载更多无焦点管理**：点击"加载更多"后，新内容追加但键盘焦点丢失 | L398-405 |
| 18 | 🟡 P1 | **空结果按钮文案不明确**："换个关键词试试"对屏幕阅读器体验不好，应改为"未找到相关商品，试试其他关键词" | L358 |
| 19 | 🟢 P2 | **价格输入框无 `inputmode`**：`el-input-number` 价格应加 `inputmode="numeric"` | L277, L285 |
| 20 | 🟢 P2 | **排序组无 ARIA**：`el-radio-group` 应加 `role="radiogroup"` + `aria-label="排序方式"` | L320 |
