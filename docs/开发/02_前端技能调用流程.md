## 第二步：逐个页面实现

每页依次执行以下 3 步循环：

| # | 操作     | 技能                | 说明                               |
| - | -------- | ------------------- | ---------------------------------- |
| 1 | 实现页面 | `frontend-design` | 粘贴下方对应页面的完整提示词       |
| 2 | UX 检查  | `ui-ux-pro-max`   | 粘贴通用 UX 检查提示词（见下方）   |
| 3 | 优化代码 | `frontend-design` | "根据上面的 UX 检查报告，优化代码" |

**通用 实现页面提示词（第 1 步用）：**

```
使用 frontend-design 技能，严格基于上面生成的 ui-design/00_MASTER.md 设计系统，实现以下页面：
- 页面1：【页面名称】
- 页面2：【页面名称】
- 要求：
  1. 严格遵守设计系统的所有规范
  2. 在规范范围内进行视觉创意发挥
  3. 添加战略性的动效和交互
  4. 生成可直接运行的代码
```

**通用 UX 检查提示词（第 2 步用）：**

```
使用 ui-ux-pro-max 技能，检查上面生成的 [页面名] 的 UX 质量：
- 对照 docs/ui-design/00_MASTER.md 设计系统检查设计一致性（颜色、字体、间距、圆角等）
- 对照 docs/design/13_前端详细设计-C端.md 对应模块检查交互流程是否完整
- 检查可访问性问题（键盘导航、焦点管理、对比度）
- 提出具体的改进建议并标注优先级（P0 必修 / P1 建议 / P2 可选）
- 将检查报告保存为 docs/design-system/[编号]_[页面名]_检查报告.md
```

**通用 优化提示词（第 3 步用）：**

```
使用 frontend-design 技能，根据上面的 UX 检查报告，优化 [页面名] 代码。
```

### 实现顺序与提示词

> 每页的 UX 检查报告保存为 `docs/ui-design/01_首页_检查报告.md`、`docs/ui-design/02_商品详情页_检查报告.md` … 以此类推。

#### ⚠️ 使用须知

- 每完成一步**复制下一步的提示词**继续发，不要一次全贴
- 提示词中 `<!-- 注释 -->` 部分是引导说明，**发送前删掉**
- 如生成的代码不完整，追加："继续生成剩余代码"
- 如 UX 检查无问题可跳过第 3 步
