### 详情

#### 1. IndexServiceImpl.rebuildIndex() 核心逻辑缺失

设计文档 §3.3 规定流程：分布式锁 → 创建新索引 → 分批灌入全量SPU → 增量回补 → 别名切换 → 保留旧版本。

当前代码只拿了锁，内部全是 TODO 注释和日志占位。**C 端搜索永远返回空结果，因为 ES 索引从未被创建或灌入数据。**

#### 2. IndexServiceImpl.syncProduct() UPSERT 未实现

DELETE 路径正常，但 UPSERT（上架/改价/改名）只记日志不写 ES。MQ 消息白白消费掉。

#### 3. IndexServiceImpl.rollback() 未实现

新索引上线后如发现搜索质量问题，无法回滚。

#### 4. SuggestServiceImpl.suggest() 未实现

completion suggester 调用完全未实现，前端搜索建议功能形同虚设。

#### 5. SearchSyncConsumer.onMessage() 空壳

消费了 `mall:search:sync` 消息但只打日志不处理。

#### 6. SpuSearchConvert.java 缺失

设计 doc §2.1 包结构图中明确定义 `convert/request/SpuSearchConvert.java`，负责 SpuSearchDTO → ProductIndexDO 转换，文件不存在。当前 IndexServiceImpl 缺少此转换器。

#### 7. 无 ES 降级逻辑

设计 doc §3.5 要求 ES 不可用时转发 `mall-product` 的 `/api/product/search/fallback` DB 兜底查询 + Redis 缓存兜底。当前 catch 直接抛 `BusinessException(ErrorCode.ES_UNAVAILABLE)`。

~~#### 8. 搜索结果用 Caffeine 本地缓存 非 Redis（已确认，排除）~~

~~设计 doc §3.2 明确要求 Redis 缓存，实际已确认使用 Caffeine 本地缓存，代码实现正确。~~

#### 9. tags 标签过滤未实现

`SearchReqDTO` 声明了 `tags` 字段但 `buildEsQuery()` 中没有任何 tag 过滤逻辑。

#### 10. ES 查询未设置超时

设计 doc §3.2 要求搜索超时 2s，但 `NativeQuery` 构建时未设置 timeout。

#### 13. 匿名路径缺失

网关配置中 `mall.security.anonymous-paths` 未包含 `/api/search/**`，导致 C 端搜索接口需要登录。搜索是公开功能，应在匿名路径中。

#### 14. Sentinel dataId 错误

Nacos `mall-search-dev.yml` 中：
```yaml
sentinel:
  datasource:
    ds1:
      nacos:
        dataId: sentinel-mall-auth  # ← 错误，应为 sentinel-mall-search
```
